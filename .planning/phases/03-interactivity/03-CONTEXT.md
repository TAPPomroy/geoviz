# Phase 3: Interactivity - Context

**Gathered:** 2026-07-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 3 adds client-side JavaScript interactivity to the HTML map generated by Phase 2. All changes are inside `BuildMapHtml()` in `mod_MapBuilder.bas` — no new VBA modules required. Deliverables: marker click popup, radius circle with neighbor highlighting, radius slider, attribute filter dropdown, and ISP legend toggle.

Phase 3 ends when: clicking a marker shows a popup, draws a radius circle, and fades non-neighbors; same-ISP neighbors are visually distinct; the slider adjusts the radius live; the attribute dropdown re-colors markers and updates the legend; and clicking a legend entry hides/shows that ISP's markers.

</domain>

<decisions>
## Implementation Decisions

### Radius Slider
- **D-01:** Unit is **miles**. Haversine function must compute distances in miles.
- **D-02:** Default radius: **0.5 miles**. Slider range: **0.1 to 5 miles**. Step granularity: Claude's discretion (suggest 0.1 mi steps).
- **D-03:** Slider sits in a **top-left Leaflet control panel** (standard Leaflet `L.control({position:'topleft'})` pattern). Panel also shows the current radius value as text (e.g., "Radius: 0.5 mi").

### Highlighting Visual Style
- **D-04:** **Selected (clicked) marker:** increase radius to 12 (from 8), fill with white (`fillColor: '#ffffff'`), keep ISP color as the border (`color: ispColor`), weight 2. The ISP color moves to the border so the marker's ISP identity is preserved.
- **D-05:** **Non-neighbor markers** (outside radius): fade to `fillOpacity: 0.15`. Normal markers remain at 0.85.
- **D-06:** **Same-ISP neighbors** (inside radius, same ISP as selected): white border (`color: '#ffffff'`), weight 3. **Other-ISP neighbors** (inside radius, different ISP): weight 1 (default). Both neighbor types retain their ISP fill color at normal opacity.
- **D-07:** The **radius circle** itself is a Leaflet `L.circle()` with a semi-transparent fill (e.g., `fillOpacity: 0.05`, `color: '#0078ff'`). Claude's discretion on exact color.

### Multi-click & Clear Behavior
- **D-08:** **Replace selection** — clicking a second company clears the previous radius circle and highlights, then activates the new selection. Only one selection at a time.
- **D-09:** **Dismiss by clicking map background** — Leaflet `map.on('click', clearSelection)` fires on empty map area. This resets all markers to normal opacity, removes the radius circle, and closes any open popup.
- **D-10:** Clicking the currently-selected marker again also clears (toggle deselect). Claude's discretion on implementation.

### Attribute Filter & Legend Interaction
- **D-11:** The attribute filter dropdown **excludes system columns**: `Lat`, `Lon`, and any column whose header matches the `HDR_LAT`, `HDR_LON`, or `HDR_TIMESTAMP` constants from `mod_Config.bas`. All other columns are included (ISP, Name, Address, and any custom attribute columns). Default selection: ISP.
- **D-12:** When the filter switches to a non-ISP column, the **legend updates** — header label changes to the selected column name, color swatches and labels reflect that column's unique values (computed the same way as ISP: sort alphabetically, assign ISP_COLORS palette, overflow → `#999999`).
- **D-13:** **Legend visibility toggles** (FILT-02): clicking an ISP legend entry hides/shows that ISP's markers. Hidden-ISP markers **remain hidden even when inside a radius** — the legend toggle state is independent of and takes precedence over radius highlighting. The hidden/shown state persists while the filter is on ISP; switching the filter column resets all visibility to shown.

### Popup Content
- **D-14:** Popup shows all JSON fields **except** `Lat` and `Lon`. Column headers (JSON keys) are displayed as-is (the column names from the Excel sheet). No reformatting of keys — what the user named the column is what appears.

### Claude's Discretion
- Exact CSS styling of the top-left control panel (background, padding, border-radius, font).
- Step size for the radius slider (suggest 0.1 mi increments).
- Exact color and opacity of the radius circle overlay.
- Whether to show a "Click a company to see neighbors" hint in the panel when no selection is active.
- Toggle-deselect implementation (D-10): single click again, or map background only as specified in D-09.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `var markers = []` — Parallel array of all CircleMarkers built in Phase 2. Phase 3 iterates this for radius highlighting without restructuring the HTML.
- `var ispColorMap = {}` / `var ispNames = []` — Already computed deterministic ISP→color mapping. Attribute filter's legend update replicates this pattern for the selected column.
- `var clusterGroup` — MarkerClusterGroup. Phase 3 interacts with individual markers from `markers[]`; clustering continues to work around selections.
- `L.circleMarker` with `setStyle()` — Leaflet CircleMarker supports `marker.setStyle({fillOpacity: 0.15})` in-place. Phase 3 uses this for fading/restoring non-neighbors.

### Established Patterns
- **AppendLine helper** — All JS is generated via `AppendLine sb, "..."` string concatenation in VBA. Phase 3 adds more JS blocks to `BuildMapHtml()` using this same pattern.
- **Inline JS in HTML** — All interactivity is in the single generated HTML file's `<script>` block. No external JS files.
- **ISP color assignment** — Sort alphabetically, index into `ISP_COLORS[7]`, overflow to `#999999`. Attribute filter legend reuses the exact same algorithm for the selected column.
- **ADODB.Stream file write** — Already handled in Phase 2's `WriteUtf8File`. Phase 3 adds no new file I/O.

### Integration Points
- Phase 3 adds JS **after** the existing marker loop (`companies.forEach(...)`) and legend in `BuildMapHtml()`. The `markers[]` array and `ispColorMap` are in scope for new click handlers.
- The **Leaflet legend** built in Phase 2 must become interactive (D-12, D-13) — Phase 3 replaces the static legend HTML with a dynamically-managed version that supports toggling.
- The marker `forEach` loop in Phase 2 does not bind popups. Phase 3 adds `.bindPopup()` or a `.on('click', ...)` handler per marker.

</code_context>

<specifics>
## Specific Ideas

- Radius range is intentionally tight (0.1–5 mi) — this is for companies in a dense urban/campus environment, not regional ISP coverage zones.
- The attribute filter dropdown defaults to ISP (the primary use case) and allows switching to any other column for alternate analysis angles.
- "All non-system columns" for the dropdown is determined by excluding known system field names (`Lat`, `Lon`, timestamp) at JS runtime — no VBA changes needed.

</specifics>
