## Summary

Phase 3 adds all client-side interactivity to the single HTML file generated by `BuildMapHtml()`. No new VBA modules are needed — the deliverable is additional JavaScript blocks appended via `AppendLine` inside the existing function.

The technical work falls into four areas: (1) marker click handlers with popup, radius circle, and neighbor highlighting; (2) the live radius slider control; (3) the attribute filter dropdown that re-colors markers and rebuilds the legend; and (4) the interactive legend with per-ISP-value show/hide toggling. All four share a small runtime state object (`selectedMarker`, `selectedISP`, `radiusCircle`, `hiddenValues`, `currentField`).

The key Leaflet primitives are: `CircleMarker.setStyle({...})` for path style, `CircleMarker.setRadius(px)` for radius, `L.circle([lat, lon], {radius: meters})` for the distance overlay, `L.control({position})` for the custom panel, and `clusterGroup.removeLayer()` / `addLayer()` for toggling marker visibility while keeping clustering intact.

**Primary recommendation:** Implement all interactivity as one contiguous JS block appended after the existing marker loop in `BuildMapHtml()`. Split into logically-named functions (`haversine`, `applySelection`, `clearSelection`, `applyFilter`, `buildLegend`) to keep each AppendLine section focused.

```javascript
// Event bindings
markers.forEach(function(marker, i) {
    marker.on('click', function(e) {
        L.DomEvent.stopPropagation(e);
        // toggle-deselect (D-10)
        if (selectedMarker === marker) { clearSelection(); return; }
        applySelection(marker, companies[i]);
    });
});
map.on('click', clearSelection);
```

### Anti-Patterns to Avoid

- **Re-creating markers on filter change:** Never remove and re-add all markers to change color. Use `marker.setStyle({fillColor: newColor})` in-place. Re-creation destroys the `markers[]` index alignment with `companies[]`.
- **Using `setStyle({radius: N})`  for CircleMarker radius:** `setStyle` changes path properties but radius for CircleMarker requires `marker.setRadius(N)`. Call them separately.
- **Calling `marker.setRadius()` inside `setStyle()`:** Not valid. They are separate Leaflet methods.
- **Flat-earth distance:** Never use simple `Math.sqrt((dlat)^2 + (dlon)^2)` for distance. Required: haversine with Earth radius in miles.
- **L.circleMarker for the radius overlay:** `L.circleMarker` radius is in PIXELS (screen-fixed). The radius overlay must use `L.circle` which takes radius in METERS and scales with zoom.
- **Forgetting `L.DomEvent.disableClickPropagation(panelDiv)` on the slider control:** Without this, clicking/dragging the slider triggers `map.on('click', clearSelection)`.
- **Checking `hiddenValues` only at render time:** Hidden-marker state must be checked both when applying radius highlighting (D-13) and when the filter switches (reset hidden state on field change).

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Geographic radius overlay | Custom polygon/polyline | `L.circle([lat,lon], {radius: meters})` | Leaflet handles projection, zoom scaling, click-through |
| Custom map control panel | Absolute-positioned div in body | `L.control({position:'topleft'}).onAdd = fn` | Leaflet positions it correctly, stacks with zoom controls, handles resize |
| Hiding markers from cluster | `marker.setStyle({opacity:0})` | `clusterGroup.removeLayer(marker)` | Invisible-but-present markers remain in cluster counts and click targets |
| Distance computation | Flat-earth (Pythagorean) | Haversine formula | 1–3% error at 50–100 km relevant for urban ISP scenarios |

**Key insight:** Leaflet provides all needed primitives (L.circle, L.control, clusterGroup.removeLayer). Phase 3 is wiring them together, not building infrastructure.

## Common Pitfalls

### Pitfall 1: `map.on('click')` fires when clicking the slider panel

**What goes wrong:** User drags the slider; `map.on('click', clearSelection)` fires; selection is immediately cleared.

**Why it happens:** Leaflet controls sit as DOM overlays but mouse events bubble through unless explicitly stopped.

**How to avoid:** Call `L.DomEvent.disableClickPropagation(containerDiv)` on the div returned from the control's `onAdd`. Also call `L.DomEvent.disableScrollPropagation(containerDiv)` to prevent scroll-wheel zoom while hovering.

**Warning signs:** Selection clears unexpectedly when interacting with the control.

---

### Pitfall 2: `setStyle` vs `setRadius` confusion

**What goes wrong:** Calling `marker.setStyle({radius: 12})` has no effect; CircleMarker keeps its old pixel radius.

**Why it happens:** `setStyle` only applies Path-level properties (color, weight, fillColor, fillOpacity, opacity, dashArray, lineCap, lineJoin). Radius is a CircleMarker property, not a Path style property.

**How to avoid:** Always use `marker.setRadius(12)` separately from `marker.setStyle({...})`.

**Warning signs:** Visual style changes (color, opacity) work but radius does not.

---

### Pitfall 3: `L.circle` radius in meters, not miles

**What goes wrong:** Developer passes miles directly to `L.circle(latlng, {radius: 0.5})`; circle is invisibly small (0.5 meters).

**Why it happens:** `L.circle` radius is in meters (geographic). `L.circleMarker` radius is in pixels. They are different classes.

**How to avoid:** Convert before passing: `var radiusMeters = sliderValue * 1609.34;`

**Warning signs:** Circle draws but is too small to see, or too large (if km were assumed).

---

### Pitfall 4: `markers[]` index misalignment after clusterGroup manipulation

**What goes wrong:** `markers[i]` no longer corresponds to `companies[i]` after some markers are removed/re-added to clusterGroup.

**Why it happens:** `markers[]` is a parallel array built during the forEach loop. `clusterGroup.removeLayer()` removes from the Leaflet layer group but does NOT change the JS array.

**How to avoid:** Never splice or reorder `markers[]`. `removeLayer`/`addLayer` only affects Leaflet rendering, not the array. The index stays stable.

**Warning signs:** Popup shows wrong company data, or haversine compares wrong coordinates.

---

### Pitfall 5: Double-quotes in VBA AppendLine strings

**What goes wrong:** JS string containing double quotes breaks VBA string literal; compile or runtime error.

**Why it happens:** VBA string literals use `"` and escape it as `""`. A JS string like `color: "#ffffff"` becomes `"color: ""#ffffff"""` inside a VBA string — confusing but necessary.

**How to avoid:** Prefer single quotes for JS strings wherever possible (JS accepts both). Reserve double quotes in JS only where HTML attributes require them (e.g., `div.innerHTML = '<span style="...">...'`).

**Warning signs:** VBA compile error "Expected end of statement" or syntax error near a JS string with double quotes.

---

### Pitfall 6: Legend toggle breaks when marker has been removed then selected

**What goes wrong:** A hidden marker's popup opens if another code path calls `marker.openPopup()` without checking hidden state.

**Why it happens:** `clusterGroup.removeLayer(marker)` removes from the map layer, but `marker` object still exists; methods like `openPopup()` don't check whether the marker is currently on the map.

**How to avoid:** In `applySelection`, always check `if (hiddenValues[company[currentField]]) return;` before applying any selection to a marker.

### Haversine in Miles [ASSUMED — standard formula, universally correct]

```javascript
// Source: standard haversine formula with Earth radius in miles
function haversine(lat1, lon1, lat2, lon2) {
    var R = 3958.8; // Earth radius in miles
    var dLat = (lat2 - lat1) * Math.PI / 180;
    var dLon = (lon2 - lon1) * Math.PI / 180;
    var a = Math.sin(dLat/2) * Math.sin(dLat/2) +
            Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
            Math.sin(dLon/2) * Math.sin(dLon/2);
    return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
}
```

### Legend Toggle (FILT-02) — hide/show by value [ASSUMED — clusterGroup API confirmed via Phase 2 code]

```javascript
// Toggle a legend entry
function toggleValue(val) {
    if (hiddenValues[val]) {
        // Show: re-add each marker whose currentField value matches
        delete hiddenValues[val];
        markers.forEach(function(m, i) {
            if (companies[i][currentField] === val) {
                clusterGroup.addLayer(m);
            }
        });
    } else {
        // Hide: remove matching markers from clusterGroup
        hiddenValues[val] = true;
        markers.forEach(function(m, i) {
            if (companies[i][currentField] === val) {
                clusterGroup.removeLayer(m);
                // If this was the selected marker, clear selection
                if (m === selectedMarker) { clearSelection(); }
            }
        });
    }
    buildLegend();  // update visual state of legend entries
}
```

## Open Questions

1. **`buildColorMap` called with non-ISP field when no selection is active**
   - What we know: `applyFilter` must re-color all non-hidden markers via `setStyle({fillColor: getFieldColor(company[field])})`
   - What's unclear: Should `applyFilter` also call `clearSelection()` to avoid a stale radius circle? D-12 doesn't specify.
   - Recommendation: Yes — always call `clearSelection()` at start of `applyFilter`. Simpler than trying to re-apply highlighting under a different color scheme.

2. **Popup bind style: `bindPopup()` vs `on('click')` with `openPopup()`**
   - What we know: Both work. `bindPopup()` auto-handles close-on-map-click; `on('click')` gives full control.
   - What's unclear: `bindPopup()` may conflict with D-09 (map click should also clear radius), because Leaflet's built-in popup close fires before our `map.on('click')`.
   - Recommendation: Use `marker.on('click')` handler that manually calls `marker.bindPopup(html).openPopup()` or `L.popup().setContent(html).setLatLng(latlng).openOn(map)`. The latter is simpler and does not leave stale popup bindings.

3. **Legend rebuild: full DOM replace vs in-place update**
   - What we know: Phase 2 sets `div.innerHTML = html` inside `legend.onAdd`. Phase 3 needs to update this dynamically.
   - Recommendation: Store a reference to the legend div (`var legendDiv = null`) in `legend.onAdd`, then `buildLegend()` replaces `legendDiv.innerHTML`. No need to call `legend.remove()` and `legend.addTo(map)`.
